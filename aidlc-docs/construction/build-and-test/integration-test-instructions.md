# 集成测试指南 — ClipSmart

## 测试场景（手动执行）

以下测试场景需在真实 macOS 设备上手动验证（单元测试使用 in-memory DB，无法测试系统级交互）。

---

### 场景 1：端到端剪切板监听与历史记录

**目的**: 验证 ClipboardMonitor → HistoryStore → UI 全链路

**步骤**:
1. 启动 ClipSmart
2. 在任意文本编辑器中选中文字并按 ⌘+C 复制
3. 按 ⌘+Shift+V 唤出面板
4. **预期**: 面板显示刚才复制的文字（应在 1 秒内出现）

---

### 场景 2：选中记录写入剪切板

**目的**: 验证 ClipItemView 单击 → HistoryViewModel.selectItem → NSPasteboard 写回

**步骤**:
1. 复制多段文字后唤出面板
2. 单击非最新一条记录
3. **预期**: 面板关闭
4. 在文本框中按 ⌘+V 粘贴
5. **预期**: 粘贴的是步骤 2 点击的那条记录内容（而非最近一次复制）

---

### 场景 3：图片复制与显示

**目的**: 验证图片类型的提取、存储和缩略图展示

**步骤**:
1. 截图（⌘+⌃+⇧+4 复制到剪切板）
2. 唤出面板
3. **预期**: 面板中显示 44×44 图片缩略图（不是文字 [图片]，缩略图应可见）

---

### 场景 4：FIFO 淘汰验证

**步骤**:
1. 打开偏好设置 → 将最大条数设为 **5**
2. 依次复制 6 段不同文字
3. 唤出面板
4. **预期**: 面板只显示 5 条，最早复制的第 1 条不见了

---

### 场景 5：固定记录不被淘汰

**步骤**:
1. 将最大条数设为 5
2. 复制 3 条，右键第 1 条 → "固定"
3. 再复制 5 条（普通区已满）
4. **预期**: 固定的那条记录仍在面板顶部

---

### 场景 6：持久化（重启后恢复）

**步骤**:
1. 复制几段内容，确认面板中可见
2. 完全退出 ClipSmart（状态栏图标右键 → "退出 ClipSmart"）
3. 重新启动 ClipSmart
4. 唤出面板
5. **预期**: 之前的历史记录仍然存在

---

### 场景 7：全局热键自定义

**步骤**:
1. 偏好设置 → 快捷键 → 点击录制区域，按新快捷键（例如 ⌃+Space）
2. 切换到其他应用
3. 按新快捷键
4. **预期**: ClipSmart 面板在任意应用前台时均可唤出

---

### 场景 8：Dock 模式切换

**步骤**:
1. 偏好设置 → 通用 → 打开"在 Dock 中显示图标"
2. **预期**: Dock 中出现 ClipSmart 图标，可通过 Cmd+Tab 切换到
3. 关闭该选项
4. **预期**: Dock 图标消失，回到纯菜单栏模式
