# 需求文档 — Mac 智能剪切板（ClipSmart）

## 意图分析摘要

| 项目 | 内容 |
|------|------|
| **用户请求** | 开发一款 macOS 原生智能剪切板管理器，支持自定义唤醒快捷键、历史记录管理、多内容类型、极简 UI |
| **请求类型** | 全新项目（Greenfield） |
| **范围估计** | 单一应用，多组件（剪切板监控、UI面板、持久化、设置） |
| **复杂度估计** | 中等偏复杂（原生 macOS 系统级权限、全局热键、多内容类型） |

---

## 技术栈决策

| 项目 | 选择 |
|------|------|
| **语言** | Swift 5.9+ |
| **UI 框架** | SwiftUI |
| **最低系统要求** | macOS 15（Sequoia）/ 理想目标 macOS 26 |
| **数据持久化** | Core Data 或 SwiftData |
| **全局热键** | KeyboardShortcuts（第三方 SPM 包）或 CGEventTap |
| **分发** | .dmg 安装包 |

## 功能需求

### FR-01：剪切板监控
- 应用在后台持续监控系统剪切板（NSPasteboard）
- 每当检测到新的复制内容，自动追加至历史记录
- 重复内容去重：若新复制内容与历史最新一条完全一致，不重复记录

### FR-02：历史记录管理
- **默认容量**: 50 条
- **最大容量**: 200 条
- **可调范围**: 用户可在设置中调整，范围 1 ~ 200
- **淘汰策略**: FIFO（先进先出）— 超出容量后，最早的非固定记录被替换
- **固定记录**: 用户可将任意记录标记为「固定（Pinned）」，固定记录不受容量限制淘汰

### FR-03：支持内容类型
| 类型 | 说明 |
|------|------|
| **纯文本** | 包括 Unicode、代码、URL、Emoji |
| **富文本** | RTF / HTML 格式文本（保留格式预览） |
| **图片** | PNG、JPEG、GIF、TIFF、WebP 等 |
| **文件引用** | 从 Finder 复制的文件/文件夹路径 |

### FR-04：全局唤醒快捷键
- 应用注册全局热键，无论当前在哪个应用均可触发
- **默认快捷键**: ⌘ + Shift + V
- 用户可在设置中自定义快捷键组合
- 再次按下快捷键或按 Esc 关闭面板

### FR-05：UI 面板
- **形式**: 浮动窗口，类 Spotlight 风格，居中显示于屏幕
- **背景**: macOS 26 液态玻璃（Liquid Glass）材质，背景高斯模糊
- 页面极简，无多余装饰
- 窗口无标题栏，圆角设计
- 显示/隐藏带有流畅动画过渡（spring 动画）
- 点击面板外区域自动关闭

### FR-06：记录交互
- **单击**: 将内容写入系统剪切板 → 关闭面板（用户手动 ⌘+V 粘贴，无需辅助功能权限）
- **右键菜单**:
  - 复制到剪切板（不关闭面板，方便连续操作）
  - 固定 / 取消固定
  - 删除该记录
- 固定记录在列表顶部显示，与普通记录有视觉区分

### FR-07：搜索功能
- 面板顶部搜索栏，支持文本内容全文搜索
- 实时过滤，边输入边搜索
- 图片类型显示缩略图，不参与文本搜索
- 搜索结果高亮匹配关键词

### FR-08：应用运行模式
- **默认**: 菜单栏应用（Menu Bar App），不出现在 Dock
- **可切换**: 用户可在设置中切换为「同时在 Dock 中显示」
- 菜单栏图标单击显示/隐藏面板，右键显示快速菜单

### FR-09：数据持久化
- 历史记录（含图片）持久保存到本地磁盘
- 应用重启、Mac 重启后历史记录继续可用
- 图片以压缩格式存储以控制磁盘占用

### FR-10：设置界面
- 通过菜单栏图标 → 「偏好设置」打开
- 设置项：
  - 全局唤醒快捷键（可录制）
  - 历史记录最大数量（滑块/步进器，范围 1-200）
  - 应用运行模式（菜单栏 / 菜单栏 + Dock）
  - 清空所有历史记录（含确认弹窗）
  - 开机自动启动（Login Item）

## 非功能需求

### NFR-01：性能
- 剪切板监控 CPU 占用 < 0.5%（空闲时）
- 面板从热键按下到显示 < 100ms
- 搜索响应时间 < 50ms（200条以内）
- 图片存储与加载使用异步处理，不阻塞 UI

### NFR-02：内存
- 应用后台运行内存 < 50MB
- 图片缩略图懒加载，按需渲染

### NFR-03：安全与隐私
- 不上传任何数据，所有数据仅保存本地
- 图片数据加密存储（使用 macOS Keychain 或文件加密）
- 应用**不需要**辅助功能（Accessibility）权限，仅写入剪切板，由用户手动 ⌘+V 完成粘贴

### NFR-04：用户体验
- UI 极简，内容优先
- 所有交互操作有视觉反馈
- 支持键盘导航（方向键选择记录，Enter 粘贴，Delete 删除）
- 深色模式 / 浅色模式自动跟随系统

### NFR-05：可维护性
- 采用 MVVM 架构
- 组件化设计，UI 与业务逻辑分离
- 代码注释完整

## 用户场景

### 场景 1：快速粘贴历史内容
用户写文档时，需要粘贴一段 10 分钟前复制过的代码。按下 ⌘+Shift+V，浮动面板出现，找到目标记录单击，内容写入剪切板且面板关闭，用户再按 ⌘+V 完成粘贴。

### 场景 2：固定常用模板
用户经常复制同一段文字（如邮件签名），通过右键将其固定，之后即使复制了很多其他内容，该记录始终在列表顶部。

### 场景 3：搜索历史图片
用户记得自己曾经复制过一张截图，在搜索栏输入不到结果（图片无法搜索），但可以通过滚动缩略图快速定位。

### 场景 4：调整历史容量
用户觉得 50 条太少，打开偏好设置，将最大历史数量调整为 150 条。

## 约束条件

- 目标平台：macOS（仅 Mac，不考虑 iOS/iPadOS）
- 不依赖云服务，完全离线运行
- .dmg 分发，需支持 Apple Silicon（arm64）和 Intel（x86_64）架构
- UI 目标风格：macOS 26 Liquid Glass 设计语言
